## 1. テスト環境の準備

### 1.1 必要なサービス

| サービス | 起動コマンド | URL |
|---------|------------|-----|
| Next.js (Frontend + BFF) | `npm run dev` | http://localhost:3000 |
| Agent Server (FastAPI) | `cd agent && python -m uvicorn alcheme.server:app --reload --port 8001` | http://localhost:8001 |

**本番環境テスト（Cloud Run デプロイ済みの場合）:**

| サービス | URL |
|---------|-----|
| Web (フロントエンド) | `https://alcheme-web-<hash>-an.a.run.app` |
| Agent (内部API) | `https://alcheme-agent-<hash>-an.a.run.app` (非公開) |

### 1.2 環境変数

- `.env.local` が設定済みであること（Firebase、Google Cloud、楽天 API、`AGENT_SERVER_URL`）
- `agent/.env` が設定済みであること（Vertex AI、Rakuten、Weather API キー、GCS バケット名）
- **特に確認**: `GOOGLE_WEATHER_API_KEY` が設定されていること（天気APIの既知バグに関連）

### 1.3 テスト用ブラウザ設定

- **推奨**: Chrome DevTools のモバイルエミュレーション（Pixel 7: 412×915）
- DevTools > Toggle Device Toolbar (Ctrl+Shift+M) > Pixel 7 を選択
- 言語設定: 日本語 (ja-JP)
- **Console タブを常時表示**: エラーが出た場合は記録すること

### 1.4 テスト用アカウント

| 用途 | メールアドレス | パスワード |
|------|-------------|-----------|
| 新規登録テスト | test123456@example.com | test123456 |
| 既存ユーザーテスト | test@example.com | password123 |

### 1.5 テスト前チェック

| # | 確認項目 | OK/NG |
|---|---------|-------|
| 1 | `npm run dev` でフロントエンドが起動するか | |
| 2 | Agent Server が `http://localhost:8001/docs` でSwagger UIが表示されるか | |
| 3 | ブラウザ DevTools のモバイルエミュレーションが有効か | |
| 4 | Console にエラーが出ていないか | |
