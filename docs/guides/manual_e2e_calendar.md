# FEAT-004 Google Calendar 連携 — 手動 E2E テストガイド

## 前提条件
- GCPセットアップ完了（`docs/guides/google-calendar-setup.md` 参照）
- `.env.local` に Calendar OAuth 環境変数設定済み
- `npm run dev` でローカルサーバー起動済み

---

## Scenario 1: Google Calendar 連携
1. 設定 → カレンダー連携 に移動
2. 「Google カレンダーを連携する」ボタンをクリック
3. Google OAuth 画面で認証を完了
4. カレンダー設定ページにリダイレクトされ「連携中」と表示されることを確認
5. カレンダー一覧がチェックボックスで表示されることを確認
6. プライマリカレンダーがデフォルトで選択されていることを確認

## Scenario 2: カレンダー選択
1. 連携済み状態でカレンダー一覧を確認
2. プライマリ以外のカレンダーにチェックを入れる
3. ページをリロード → 選択が保持されていることを確認

## Scenario 3: 今日のイベントプレビュー
1. Google Calendar に今日のイベントを登録（例: 14:00 会議）
2. カレンダー設定ページで「今日の予定」セクションにイベントが表示されることを確認
3. 時刻・場所が正しく表示されることを確認

## Scenario 4: 手動スケジュール入力
1. Google Calendar を連携**しない**状態で
2. カレンダー設定ページの「予定を手動入力」に「午後3時から会議、夜はディナー」と入力
3. 保存
4. チャットで「今日の予定に合わせてメイクして」と送信
5. AI が手動入力の予定を使用してメイク提案することを確認

## Scenario 5: TPO Tactician での利用
1. Google Calendar 連携済み + 今日のイベントあり
2. チャットで「天気と予定に合わせてメイクして」と送信
3. TPO Tactician が Google Calendar のイベントを使用して戦略を生成することを確認
4. レシピに予定の TPO（フォーマル/カジュアル等）が反映されていることを確認

## Scenario 6: 連携解除
1. 連携済み状態で「連携を解除」をクリック
2. ステータスが「未連携」に変わることを確認
3. カレンダー一覧・イベントプレビューが消えることを確認
4. チャットで TPO 提案を依頼 → 予定なし or 手動入力にフォールバックすることを確認

## Scenario 7: OAuth 拒否
1. 連携フローで Google 認証画面で「拒否」をクリック
2. エラーメッセージが表示されることを確認
3. 連携状態が変わらないことを確認
