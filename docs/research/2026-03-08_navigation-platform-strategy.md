## 1. 現状分析: alche:me のナビ構造の問題

### ボトムナビ（現在）

| 位置 | タブ | 方向性 |
|------|------|--------|
| 1 | AI美容部員 `/chat` | 自分 |
| 2 | My Cosme `/inventory` | 自分 |
| 3 | + (Quick Action) | アクション |
| 4 | フィード `/feed` | みんな |
| 5 | マイページ `/mypage` | 自分 |

**自分:みんな = 4:1** — 「ツール」の構造

### サイドメニュー（現在）の問題

- MAIN_MENU: AI美容部員, My Cosme, レシピ, メイク日記
- SUB_MENU: マイページ, みんなのコスメ, レシピ一覧, 設定
- **問題1**: レシピ と レシピ一覧 が同じ `/recipes` への完全重複
- **問題2**: AI美容部員, My Cosme, マイページ がボトムナビと重複（8項目中3つが無駄）
- **問題3**: サイドメニューの固有価値は実質4項目だけ（レシピ, 日記, コスメ, 設定）

## 3. 「ECの死」— Agentic Commerce の台頭

### 日経記事の核心

「SaaSの死」に続く「ECの死」— 買い物エージェントの破壊力（日経 2026年2月）

**従来EC（死にゆくモデル）:**
```
ユーザー → 検索 → 商品一覧 → 比較 → カート → 購入
         ↑ 人間のアテンション（注意力）を広告主に売るビジネス
```

**Agentic Commerce（新モデル）:**
```
ユーザー → 目的を伝える → AIが選び → 提案 → ワンタップ購入
         ↑「便利さ」と「パーソナライズ」で利益を生むビジネス
```

### 主要数字

| 指標 | 数値 | 出典 |
|------|------|------|
| Agentic Commerce 市場予測 (2030年) | **$3〜5兆** | McKinsey (2025年10月) |
| 2025年ホリデー AI経由EC流入 | 前年**2倍** | Retail業界レポート |
| AI関連小売売上 (2025年ホリデー) | **$262B** | 同上 |
| AI参照トラフィック (Amazon/Walmart) | 最大**25%** | 同上 |

### 主要プレイヤーの動き

| プレイヤー | 動き | 時期 |
|-----------|------|------|
| **Walmart「Sparky」** | 目的ベース（「バーベキューしたい」→ 食材+器具+レシピ一括提案） | 2025年6月 |
| **OpenAI「Instant Checkout」** | ChatGPT内で直接購入完結（Stripe連携） | 2026年2月 |
| **Google + Shopify** | Universal Commerce Protocol (UCP) 発表 | 2026年1月 |
| **L'Oréal「Beauty Genius」** | セルフィー → 肌診断 → 商品提案の一気通貫 | 2025年〜 |
| **アテニア「AI美容部員」** | 音声+チャットで対面接客の寄り添いをオンライン化 | 2026年2月 |
| **Tencent Beauty AI** | マルチエージェント構成で肌分析→推薦→購入 | 2025年〜 |

### 「勝者の条件」

記事が指摘する勝者の3条件:

1. **ユーザーの文脈を深く知っている** — 購入履歴だけでなく、使用状況・好み・ライフスタイル
2. **目的ベースで提案できる** — 「商品を探す」ではなく「目的を達成する」
3. **そのまま購入まで完結できる** — 提案からワンタップで購入

## 4. alche:me の独自優位性

### 他社が持たないデータ

```
① 在庫データ（何を持っているか + 残量）
   ×
② 使用データ（何をどう使っているか + 頻度 + 評価）
   ×
③ パーソナルデータ（肌質 + パーソナルカラー + 好みのスタイル）
   ×
④ AI美容部員（目的→ソリューション変換エンジン）
   =
⑤ 「今持っているもので何ができるか」+ 「何を買えば良いか」
```

**Walmart は「何を買ったか」しか知らない。**
**Amazon は「何を検索したか」しか知らない。**
**LIPS は「何をレビューしたか」しか知らない。**
**alche:me は「何を持っていて、何をどう使っていて、何が足りないか」を知っている。**

### ポジショニングマップ

```
         パーソナライズ度
              ↑
    高い │  ★ alche:me
         │  (在庫+使用+肌質データ)
         │
         │      ● L'Oréal Beauty Genius
         │      (肌分析のみ)
         │
         │  ● LIPS
         │  (レビューベース)
         │
    低い │  ● Amazon
         │  (購入履歴のみ)
         └──────────────────────→
           検索型              ソリューション型
           (商品を探す)        (目的を達成する)
```

alche:me は「パーソナライズ × ソリューション型」の**右上象限**。

## 5. バイラルループ設計

### 現状の問題

alche:me にはバイラルループの**素材は全部ある**が、ナビ構造が「自分で完結」するため、ループが回らない。

### あるべきバイラルループ

```
① ユーザーがAIでメイクレシピを生成（コンテンツ生産）
    ↓
② フィードに投稿 → 他ユーザーが見て「やりたい」（発見・欲求）
    ↓
③ 「このレシピを再現」→ AIが手持ちで代替提案（ソリューション）
    ↓
④ 不足コスメを購入（コマース）→ メイク記録を投稿（①に戻る）
```

## 6. 提案: alche:me のあるべき姿

### 設計哲学

> **「自分を知って、みんなに広げて、賢く買う」**

alche:me の本質的強みは **在庫 × 使用 × パーソナルデータ** の三位一体。
しかし現在のナビゲーションはこのデータを「自分のため」にだけ使っている。
プラットフォームへ進化するには、このデータを **「みんなのため」「購買のため」** にも活用する導線が必要。

### 6.1 ボトムナビ（新構造）

```
旧: AI美容部員 | My Cosme | ＋ | フィード | マイページ
     自分       自分     アクション 外向き   自分
     (4:1 = ツール)

新: ホーム | 発見 | AI美容部員 | ショップ | マイページ
     外向き  外向き  コア強み    外向き    自分
     (3:1:1 = プラットフォーム)
```

| 位置 | タブ | アイコン | 内容 | 方向性 |
|------|------|---------|------|--------|
| 1 | **ホーム** | Home | フィード + みんなのレシピ + トレンド | 外向き（発見） |
| 2 | **発見** | Search | みんなのコスメ + ランキング + ブランド検索 | 外向き（発見） |
| 3 | **AI美容部員** | Sparkles | チャット（alche:me のコア） | コア機能 |
| 4 | **ショップ** | ShoppingBag | Next Cosme 進化形 → 目的ベース購入提案 | 外向き（購買） |
| 5 | **マイページ** | User | プロフィール + 統計 | 自分 |

**設計意図:**
- AI美容部員はセンターに配置 — **LIPS が＋を消してショップにしたように、alche:me は AI をセンターに据える**
- 「ホーム」と「発見」で**外向きの導線を2倍に**
- 「ショップ」で **Agentic Commerce への伏線**を張る
- 「＋」は FAB（フローティングアクションボタン）に移動

### 6.2 FAB（フローティングアクションボタン）

```
画面右下に常時表示:

  ┌────┐
  │ ＋ │  ← グラデーション丸ボタン
  └────┘

タップ → QuickActionSheet（既存）を展開
```

LIPS と同じパターン: ボトムナビの＋を FAB に昇格。
ボトムナビの1枠を「ショップ」に譲る。

### 6.3 サイドメニュー（新構造）

**原則: ボトムナビとの重複ゼロ。サイドメニュー固有の価値のみ。**

```
┌─────────────────────────┐
│ 👤 ユーザー名            │
│    @email               │
├─────────────────────────┤
│ 💄  My Cosme            │  ← 在庫管理（自分の棚）
│ 📖  レシピ              │  ← レシピ一覧
│ 📅  メイク日記           │  ← Beauty Log
│ 📆  カレンダー           │  ← Google Calendar 連携
├ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┤
│ ⚙️  設定                │
│ 🚪  ログアウト           │
└─────────────────────────┘
```

| 項目 | ボトムナビとの関係 |
|------|-------------------|
| My Cosme | ボトムナビから削除 → サイドメニューに移動 |
| レシピ | サイドメニュー固有（重複解消: レシピ一覧を削除） |
| メイク日記 | サイドメニュー固有 |
| カレンダー | サイドメニュー固有（新規追加） |
| 設定 | サイドメニュー固有 |

**削除項目と理由:**
- ~~AI美容部員~~ → ボトムナビで常時アクセス可
- ~~マイページ~~ → ボトムナビで常時アクセス可
- ~~レシピ一覧~~ → レシピと完全重複（同一 /recipes）
- ~~みんなのコスメ~~ → 「発見」タブに統合

### 6.4 ホーム画面の構成

```
┌─────────────────────────────┐
│ ☰  [alche:me]    🔔         │ ← トップバー
├─────────────────────────────┤
│ おすすめ | フォロー中 | 新着  │ ← サブタブ
├─────────────────────────────┤
│ 🔮AI診断  💄ランキング       │ ← アイコングリッド（横スクロール）
│ 📖レシピ  🆕新作コスメ       │    ショートカット
├─────────────────────────────┤
│                             │
│   レシピカード / メイク投稿    │ ← コンテンツフィード
│   （みんなの投稿）           │    カード形式
│                             │
│                        [＋] │ ← FAB
└─────────────────────────────┘
```

### 6.5 「ショップ」タブ — Agentic Commerce への進化

**Phase 1（現行 Next Cosme の発展形）:**
```
┌─────────────────────────────┐
│ 🛍️ ショップ                 │
├─────────────────────────────┤
│ "春メイクに挑戦したい"       │ ← 目的入力
│ [AIに相談する]              │
├─────────────────────────────┤
│ 🔥 あなたへのおすすめ        │
│ ┌───┐ ┌───┐ ┌───┐         │ ← AI提案カード
│ │   │ │   │ │   │         │   在庫データから
│ │ 💄│ │ 💄│ │ 💄│         │   「足りないもの」を提案
│ └───┘ └───┘ └───┘         │
├─────────────────────────────┤
│ 📋 Next Cosme リスト        │ ← 従来の欲しいものリスト
│ ┌─────────────────────┐    │
│ │ NARS ブラッシュ       │    │
│ │ ★ 購入する →         │    │
│ └─────────────────────┘    │
└─────────────────────────────┘
```

**Phase 2（Agentic Commerce 本格化）:**
- 「春メイクしたい」→ AI が在庫を確認 → 持っているもの + 必要なもの + 購入リンク を一括提案
- ワンタップ購入（楽天アフィリエイト連携）
- 目的ベース提案: 商品を探すのではなく、**目的を達成する**

### 6.6 バイラルループの実装

```
① ユーザーが AI でメイクレシピを生成（コンテンツ生産）
    ↓
② ホーム・フィードに投稿 → 他ユーザーが見て「やりたい」（発見・欲求）
    ↓
③ 「このレシピを再現」→ AI が手持ちで代替提案（ソリューション）
    ↓
④ 不足コスメを「ショップ」で購入 → メイク記録投稿（①に戻る）
```

**ナビゲーションが回すループ:**
- ホーム → レシピ発見 → AI 相談 → ショップで購入 → 自分のレシピを投稿 → ホーム

### 6.7 実装フェーズ

| Phase | 内容 | 工数目安 |
|-------|------|---------|
| **Phase A** | サイドメニュー整理（重複削除 + カレンダー追加） | 小 |
| **Phase B** | ボトムナビ再構築（ホーム・発見・AI・ショップ・マイページ） | 中 |
| **Phase C** | FAB 実装（＋ボタンを右下フローティングに） | 小 |
| **Phase D** | ホーム画面リデザイン（アイコングリッド + フィード統合） | 中 |
| **Phase E** | ショップタブ Phase 1（目的入力 + AI提案 + Next Cosme 統合） | 大 |
| **Phase F** | Agentic Commerce Phase 2（ワンタップ購入 + 購入リンク一括） | 大 |

**推奨順序: A → C → B → D → E → F**
（サイドメニュー整理 → FAB は小規模で即効性が高い）

### 6.8 ビフォー/アフター サマリー

| | 現在 | 提案後 |
|---|------|--------|
| **ボトムナビ** | AI / My Cosme / ＋ / フィード / マイページ | ホーム / 発見 / AI美容部員 / ショップ / マイページ |
| **自分:外向き比率** | 4:1 | 1:3:1 |
| **サイドメニュー項目** | 8（うち重複4） | 6（重複0） |
| **コマース導線** | なし | ショップタブ + AI提案 |
| **バイラルループ** | 素材はあるが未接続 | ナビが回すループ構造 |
| **＋ボタン** | ボトムナビ中央 | FAB（右下フローティング） |
| **ポジション** | パーソナルツール | Agentic Beauty Platform |
