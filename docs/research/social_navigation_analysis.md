## 1. TikTok ナビゲーション分析

### 1.1 タブ構成概要

TikTok は **5タブ構成** のボトムナビゲーションを採用し、中央に突出した「+」ボタンを配置するパターンの代表格である。

| タブ位置 | ラベル | アイコン | 機能概要 |
|---------|--------|---------|---------|
| 1（左端） | Home | 家アイコン (outlined/filled) | For You Page + Following フィード |
| 2 | Discover / Friends | 虫眼鏡 / 人物アイコン | トレンド検索 / 友達フィード（地域により異なる） |
| 3（中央） | **+** (Create) | 「+」記号を囲むピルボタン | 動画撮影・写真投稿・LIVE配信 |
| 4 | Inbox | 吹き出しアイコン | DM・アクティビティ通知 |
| 5（右端） | Profile | 人物アイコン | マイページ・投稿一覧・設定 |

### 1.2 中央「+」ボタン詳細仕様

TikTok の最も特徴的な UI 要素は、中央の Create ボタンである。

| パラメータ | 値 | 備考 |
|-----------|-----|------|
| ボタン形状 | ピル型（角丸長方形） | 他タブのアイコンと明確に区別 |
| ボタンサイズ | 約 48pt x 30pt | 他のアイコン（24x24pt）の **約1.5〜2倍** |
| 視覚的突出 | タブバーより **約4〜6pt 上方** に突出 | 物理的な高さで強調 |
| 背景色（ダーク） | 白背景 + 左にシアン、右にマゼンタのレイヤー | ブランドカラーのアクセント |
| 背景色（ライト） | 黒背景 + 左にシアン、右にマゼンタ | テーマに応じた反転 |
| アイコン | 「+」記号（太字、白/黒） | ラベルなし -- アイコンのみ |
| タップエリア | 最低 44x44pt (Apple HIG 準拠) | 実際のタップ領域はボタン外縁まで拡張 |
| アニメーション | タップ時に scale 0.95 → 1.0 のバウンス | 触感フィードバック (haptic) 併用 |

#### Create フロー

「+」ボタンをタップすると、以下のいずれかの挙動となる:

1. **直接カメラ起動**（デフォルト）: 撮影画面に遷移
2. **ボトムシート**（長押し / 上スワイプ）: 複数オプションを表示

| オプション | アイコン | 説明 |
|-----------|---------|------|
| 動画撮影 | カメラアイコン | 最大10分の動画（セグメント撮影対応） |
| 写真投稿 | 画像アイコン | フォトモード（複数枚スライドショー） |
| LIVE配信 | ライブアイコン | リアルタイム配信（条件付き解放） |
| テンプレート | グリッドアイコン | トレンドテンプレートから選択 |
| ギャラリー選択 | フォルダアイコン | 端末内の既存メディアから選択 |

### 1.3 タブバー仕様

| パラメータ | 値 |
|-----------|-----|
| タブバー高さ | 49pt（Safe Area Inset 除く） |
| Safe Area Inset 含む全体高さ | 83pt（iPhone X 以降） |

| 背景色（ライトテーマ） | #FFFFFF (白) |

| アイコン〜ラベル間隔 | 2pt |
| アクティブ状態色 | #FFFFFF（ダーク時）/ #000000（ライト時） |

| アクティブ時アイコンスタイル | Filled（塗りつぶし） |
| 非アクティブ時アイコンスタイル | Outlined（輪郭のみ） |

### 1.4 For You Page (FYP) のフィード構造

| 項目 | 仕様 |
|------|------|
| スクロール方式 | **Vertical Snap-Scroll**（縦方向にスナップ） |
| 1画面 = 1コンテンツ | 全画面表示（ステータスバー・タブバー含む） |
| スワイプジェスチャー | 上スワイプ: 次の動画 / 下スワイプ: 前の動画 |
| 横スワイプ | 投稿者プロフィールへ遷移 |
| オーバーレイ UI | 右サイドに縦並びアクション: プロフィール / いいね / コメント / シェア / 音楽 |
| フィードタブ切り替え | 上部に「Following」「For You」の2セグメント |
| アルゴリズム | 視聴時間・完走率・エンゲージメント率に基づくレコメンデーション |
| プリロード | 次の2〜3動画を先行バッファリング |

#### Vertical Snap-Scroll の技術仕様

```
+---------------------------+
|  Status Bar (44pt)        |
|---------------------------|
|  Following | For You      |  <-- セグメントコントロール
|                           |
|                           |
|    FULL-SCREEN VIDEO      |  <-- 動画が画面全体を占有
|                           |
|          ♡ 12.3K          |  <-- 右サイド オーバーレイ
|          💬 456           |
|          -> Share         |
|          ♪                |
|                           |
|  @username  #hashtag...   |  <-- 左下 オーバーレイ
|---------------------------|
|  🏠  🔍  [+]  💬  👤    |  <-- ボトムタブバー (49pt)
+---------------------------+
```

### 1.5 Discover / 検索タブ

| 要素 | 仕様 |
|------|------|
| 検索バー | 上部固定、角丸テキストフィールド（高さ 36pt） |

### 1.6 Profile タブ

| 要素 | 仕様 |
|------|------|

| 設定アクセス | 右上のハンバーガー or 歯車アイコン |

## 2. Instagram ナビゲーション分析

### 2.1 ナビゲーション進化の歴史 (2010〜2024)

Instagram のタブバーは、アプリ戦略の変遷を如実に反映してきた。以下にその歴史的変遷を示す。

| 時期 | タブ構成 | 変更理由 |
|------|---------|---------|
| **2010〜2016 初期** | Feed / Explore / Camera / Activity / Profile | 写真共有SNSとしてのシンプルな設計 |
| **2016 Stories 導入** | Feed / Explore / Camera(+) / Activity / Profile | Stories 追加に伴うカメラタブの強調 |
| **2018〜2019** | Home / Explore / Camera(+) / Activity / Profile | IGTV の追加（Explore 内に統合） |
| **2020 大改編** | Home / Explore / **Reels** / **Shop** / Profile | Shopping タブ新設 + Reels タブ追加。Activity は通知ベルに縮小 |
| **2022 Reels 強調期** | Home / Explore / **Reels(+中央)** / Shop / Profile | 全画面 Reels をフルフィードに昇格 |
| **2023 ショップ撤去** | Home / Explore / **Create(+)** / Reels / Profile | **Shop タブ削除** -- ユーザー反発を受けた戦略転換 |
| **2024〜現在** | Home / Explore / **Create(+)** / Reels / Profile | Create を中央に再配置、Reels は専用タブ維持 |

#### 重要な教訓: Shopping タブの追加と撤去

Instagram の Shopping タブ導入と撤去は、モバイルナビゲーション設計における最も重要な事例研究の一つである。

| 観点 | 詳細 |
|------|------|
| 導入時期 | 2020年11月 |
| 配置 | タブバー第4位置（Activity の代わり） |
| ユーザー反応 | **強い反発** -- 「SNSで買い物はしたくない」「Activity通知が見つからない」 |
| ビジネス成果 | ショッピング利用率は期待を大幅に下回った |
| 撤去時期 | 2023年2月 |
| **教訓** | ビジネス目標（EC推進）をタブバーに強制すると、ユーザー体験を損なう。タブはユーザーの **行動頻度** に基づいて配置すべき |
| **alche:me への示唆** | 「Next Cosme（おすすめ購入）」を独立タブにする設計は、Instagram の失敗と同じ轍を踏むリスクがある |

### 2.2 現在のタブ構成詳細 (2024〜)

| タブ位置 | ラベル | アイコン (Inactive) | アイコン (Active) | 機能 |
|---------|--------|--------------------|--------------------|------|
| 1（左端） | Home | 家 (outlined) | 家 (filled) | フィード + Stories トレイ |
| 2 | Explore | 虫眼鏡 (outlined) | 虫眼鏡 (filled) | 検索 + モザイクグリッド |
| 3（中央） | Create (+) | 「+」角丸正方形 | -- (遷移するため) | 投稿作成フロー |
| 4 | Reels | 再生アイコン (outlined) | 再生アイコン (filled) | 短尺動画フィード |
| 5（右端） | Profile | ユーザーアバター (小) | アバター + リング | マイページ + 設定 |

### 2.3 タブバー仕様

| パラメータ | 値 |
|-----------|-----|
| タブバー高さ | 49pt（Safe Area 除く） |
| Safe Area Inset 含む全体高さ | 83pt（iPhone X 以降） |
| 背景色 | #FFFFFF（ライト） / #000000（ダーク） |
| 背景ブラー | なし（不透明背景） |
| 上端ボーダー | 0.5pt, #DBDBDB（ライト） / rgba(255,255,255,0.15)（ダーク） |
| アイコンサイズ | 24x24pt |
| タブラベル | **非表示**（アイコンのみ） |
| アクティブ時 | **Filled アイコン**（塗りつぶし） |
| 非アクティブ時 | **Outlined アイコン**（輪郭のみ） |
| Profile タブ | ユーザーのアバター画像（22pt 丸）。アクティブ時は 1.5pt の黒リングが付く |
| Create ボタン | 24x24pt の角丸正方形内に「+」。他タブより若干大きい視覚的重み |
| バッジ通知 | Activity（ハート）タブの右上に赤丸ドット（6pt） |

### 2.4 Stories パターン詳細仕様

Instagram Stories は、ephemeral content（消える投稿）の最も成功した UI パターンであり、多くのアプリに影響を与えている。

#### Stories トレイ（フィード上部）

| パラメータ | 値 |
|-----------|-----|
| 配置 | Home タブのフィード最上部 |
| スクロール方向 | **水平スクロール** |
| トレイ高さ | 約 96pt（アイコン + ラベル含む） |
| 左端要素 | 「Your Story」（自分のアバター + 青い「+」バッジ） |
| 並び順 | 未読 → 閲覧済み / アルゴリズムによるランキング |
| 下部ボーダー | 0.5pt, #DBDBDB（ライト時） |

#### Stories サークル（個別アバター）

| パラメータ | 値 |
|-----------|-----|
| 外側リング直径 | **62pt** |
| リング幅 | **2pt** |
| リングのスタイル（未読） | **グラデーション**: #F58529 (オレンジ) → #DD2A7B (ピンク) → #8134AF (パープル) |
| リングのスタイル（閲覧済み） | #C7C7CC（グレー） |
| 内側アバター直径 | **56pt** |
| アバター〜リング間ギャップ | **1pt**（白スペース） |
| ユーザー名ラベル | 10pt, SF Pro Text Regular, 最大6文字 + 省略 |
| ラベル色 | #262626（ライト）/ #FFFFFF（ダーク） |
| アイテム間スペーシング | 12pt |
| 左右パディング | 12pt |

#### Stories ビューアー

| パラメータ | 値 |
|-----------|-----|
| 表示形式 | **フルスクリーンモーダル** |
| プログレスバー | 上部に **セグメント分割バー**（Stories 数に応じて分割） |
| バー高さ | 2pt |
| バー色（進行中） | #FFFFFF |
| バー色（未再生） | rgba(255,255,255,0.35) |
| 画像自動進行 | **5秒** |
| 動画自動進行 | **動画の長さ**（最大60秒） |
| タップ操作 | 右タップ: 次 / 左タップ: 前 / 長押し: 一時停止 |
| スワイプ操作 | 左スワイプ: 次のユーザー / 右スワイプ: 前のユーザー / 下スワイプ: 閉じる |
| 返信エリア | 下部にテキスト入力バー + 絵文字リアクション |
| ヘッダー | アバター (32pt) + ユーザー名 + 投稿時刻 + 「...」メニュー |

### 2.5 Explore ページ（検索タブ）

Explore ページは Instagram の **発見エンジン** であり、モザイクグリッドレイアウトが特徴。

#### グリッドレイアウト仕様

| パラメータ | 値 |
|-----------|-----|
| 基本パターン | **1大 + 4小** のモザイクグリッド（3列ベース） |
| 繰り返し周期 | 3行ごとにパターン反転 |
| 小セルサイズ | 画面幅の **1/3** |
| 大セルサイズ | 画面幅の **2/3**（2列x2行 を占有） |
| セル間ギャップ | 1pt（ほぼシームレス） |
| セルアスペクト比 | 1:1（正方形） |
| 大セル位置 | パターンA: 右上 / パターンB: 左下 |
| 動画インジケーター | 右上に再生アイコン (Reels) |
| 複数投稿インジケーター | 右上にカルーセルアイコン |

#### グリッドパターン図解

```
パターン A (行1-3):           パターン B (行4-6):
+-----+-----+-----------+    +-----------+-----+
|  S  |  S  |           |    |           |  S  |
+-----+-----+     L     |    |     L     +-----+
|  S  |  S  |           |    |           |  S  |
+-----+-----+-----------+    +-----------+-----+
|  S  |  S  |     S     |    |  S  |  S  |  S  |
+-----+-----+-----------+    +-----+-----+-----+

S = Small cell (1/3 width)
L = Large cell (2/3 width x 2 rows)
```

#### 検索 UI

| 要素 | 仕様 |
|------|------|
| 検索バー | 上部固定、高さ 36pt、角丸 10pt、グレー背景 |
| プレースホルダー | 「検索」 + 虫眼鏡アイコン |
| 検索中状態 | キーボード表示 + 「アカウント / 音声 / タグ / 場所」タブ |
| カテゴリチップ | 検索バー下に横スクロール（IGTV / Shop / Travel / Architecture 等） |

## 3. 共通パターンの抽出

### 3.1 業界標準パターン一覧

TikTok と Instagram の分析、および先行レポート（LIPS / @cosme）の知見を統合し、モバイルアプリにおけるナビゲーションの業界標準パターンを以下に整理する。

| # | パターン | TikTok | Instagram | LIPS | @cosme | 業界普及率 |
|---|---------|--------|-----------|------|--------|-----------|
| 1 | **ボトムタブバー 5項目** | Yes | Yes | Yes | Yes | 非常に高い |
| 2 | **右端にプロフィール/マイページ** | Yes | Yes | Yes | Yes | **ほぼ100%** |
| 3 | **中央に「+」アクションボタン** | Yes | Yes | -- (FAB) | -- | 高い |
| 4 | **Filled/Outlined 切替** (Active/Inactive) | Yes | Yes | Yes | Yes | 非常に高い |
| 5 | **タブバー高さ 49pt** | Yes | Yes | -- | -- | iOS 標準 |
| 6 | **バッジ通知（赤丸ドット）** | Yes | Yes | Yes | Yes | 非常に高い |
| 7 | **スワイプジェスチャーでセクション切替** | Yes | Yes | 部分的 | 部分的 | 高い |
| 8 | **ラベル非表示（アイコンのみ）** | No (ラベルあり) | Yes | No | No | 分かれる |

### 3.2 パターン詳細分析

#### パターン1: ボトムタブバー 5項目

Apple Human Interface Guidelines は「3〜5個」を推奨。主要 SNS はすべて **5個** で統一されている。

| 理由 | 説明 |
|------|------|

| 情報設計 | 5つの「世界」でアプリの全機能を表現可能 |
| 差別化余地 | 中央1つをアクションボタンに充てても、4つのナビゲーション先を確保 |

## 6. Explore 風レシピグリッド

### 6.1 概要

Recipe タブを Instagram Explore ページのモザイクグリッドレイアウトで再設計する。従来のリスト表示に代わり、視覚的に豊かなブラウジング体験を提供する。

### 6.2 モザイクグリッドレイアウト仕様

| パラメータ | 値 | 参照 |
|-----------|-----|------|
| 列数 | **3列** | Instagram Explore |
| グリッドパターン | **1大 (2x2) + 4小 (1x1)** の繰り返し | Instagram Explore |
| 繰り返し周期 | 3行（6セル分） | Instagram |
| セル間ギャップ | **2pt** | Instagram: 1pt、alche:me は少し広め |
| 小セル幅 | `(screen_width - 4pt) / 3` | 3列均等分割 |
| 小セル高さ | 小セル幅と同一（1:1 正方形） | Instagram 準拠 |
| 大セル幅 | 小セル幅 x 2 + 2pt (ギャップ分) | 2列占有 |
| 大セル高さ | 小セル高さ x 2 + 2pt (ギャップ分) | 2行占有 |
| 角丸 | **8pt** | alche:me のデザインシステムに合わせる |
| 大セル配置パターン | 奇数ブロック: 右上 / 偶数ブロック: 左下 | Instagram |
