## 1. エグゼクティブサマリー

### 両アプリの位置づけ

| 軸 | LIPS | @cosme | alche:me（目指す姿） |
|----|------|--------|---------------------|
| **コアバリュー** | 「リアルなクチコミで出会う」 | 「日本最大級のコスメDB + クチコミ」 | 「手持ちコスメで、まだ見ぬ私に出会う」 |
| **ユーザー行動** | 発見 → レビュー → 購入 | 検索 → 比較 → 購入 | 管理 → 組み合わせ発見 → 活用 → (購入) |
| **主要ユーザー層** | 10代後半〜20代前半（Z世代） | 20代〜40代（幅広い） | 20代〜30代コスメ愛好家 |
| **収益モデル** | 広告 + EC連携 + ブランドタイアップ | 広告 + @cosme STORE(EC) + B2B | AI Premium + B2Bデータ + EC連携 |
| **ポジション** | SNS寄り（コミュニティファースト） | DB寄り（情報ファースト） | ツール寄り（パーソナルAIファースト） |

### 重要な差別化ポイント

LIPS と @cosme はいずれも **「新しいコスメに出会う場所（Buying）」** に最適化されている。alche:me は **「持っているコスメを愛する場所（Utilizing）」** という独自のポジションを取る。この分析では、両アプリの優れた UI/UX パターンのうち、alche:me のコンテキストで応用可能なものを特定する。

## 2. LIPS 詳細分析

### 2.1 アプリ概要

LIPS（リップス）は、AppBrew 社が運営する日本最大級のコスメ・美容のクチコミアプリ。累計ダウンロード数は850万以上（2024年時点）。10代後半〜20代前半の女性を中心に、リアルなクチコミと写真でコスメの発見・比較ができるプラットフォーム。

### 2.2 タブ構成とナビゲーション

#### ボトムナビゲーション（5タブ構成）

| タブ | アイコン | 機能 |
|------|---------|------|
| **ホーム** | 🏠 | フィード（おすすめ/フォロー/新着） |
| **ランキング** | 🏆 | カテゴリ別ランキング |
| **検索** | 🔍 | 商品・ブランド・投稿検索 |
| **通知** | 🔔 | いいね・コメント・フォロー通知 |
| **マイページ** | 👤 | プロフィール・投稿・クリップ・購入履歴 |

#### ナビゲーション設計の特徴

- **5タブ構成**はモバイルUXのベストプラクティスに沿っている（Apple HIG 推奨の3〜5個）
- **ホームタブ**のフィードには「おすすめ」「フォロー中」「新着」のセグメントコントロール（横スワイプ切り替え）
- 投稿ボタンは画面右下の **FAB（Floating Action Button）** として常時表示
- **ヘッダーにブランドロゴ**を配置し、ブランドアイデンティティを強調
- タブ切り替え時のアニメーションは iOS ネイティブに準拠（スムーズな遷移）

### 2.3 商品カタログ / ブラウジングパターン

#### カテゴリ階層

```
大カテゴリ（ベースメイク / アイメイク / リップ / チーク / スキンケア / etc.）
  └─ 中カテゴリ（ファンデーション / 下地 / コンシーラー / etc.）
      └─ 商品一覧（ブランド × 商品名）
          └─ 商品詳細ページ
              └─ 色番（カラーバリエーション）
```

#### ブラウジング UI パターン

1. **ランキングリスト**: カテゴリ別の人気ランキング。横スクロールのカテゴリチップ + 縦スクロールの商品リスト
2. **商品カード**: 正方形のサムネイル + 商品名 + ブランド名 + 評価星 + クチコミ数。カード型でタップ可能
3. **カラーバリエーション**: 商品内の色番が丸いカラーチップとして並び、タップで絞り込み可能
4. **スワッチ写真**: ユーザー投稿のスウォッチ（腕や唇への発色写真）が商品ページで最も重要な要素

### 2.4 検索・フィルター UX

#### 検索機能

- **インクリメンタルサーチ**: 入力中にリアルタイムで商品・ブランド・投稿が候補表示
- **トレンド検索ワード**: 検索画面を開いた時点で「いま人気のキーワード」が表示（ハッシュタグ風）
- **検索カテゴリ切り替え**: 「商品」「投稿」「ユーザー」タブでフィルタリング
- **最近の検索履歴**: 直前の検索ワードを保存・表示

#### フィルター

- **カテゴリフィルター**: 横スクロールのピルボタン（チップ）
- **ブランドフィルター**: アルファベット/50音順の一覧 + 検索
- **価格帯フィルター**: スライダー or プリセット（プチプラ / ミドル / デパコス）
- **評価フィルター**: 星評価（3.0以上、4.0以上、etc.）
- **年代フィルター**: 投稿者の年代で絞り込み（10代/20代前半/20代後半/30代/etc.）
- **肌質フィルター**: 乾燥肌/脂性肌/混合肌/敏感肌

> **alche:me への示唆**: 肌質・年代によるフィルタリングは、LIPS の大きな差別化要因。alche:me でもユーザープロフィール（肌質、パーソナルカラー）に基づいたフィルタリングを AI レシピ検索に応用できる。

### 2.5 クチコミ・投稿機能（日記的要素）

#### 投稿の構造

```
投稿 = {
  商品タグ（複数可）,
  評価（★1〜5）,
  テキスト（自由記述）,
  写真（最大10枚）,
  ハッシュタグ,
  パーソナルデータ表示（肌質、年代、パーソナルカラー等）
}
```

#### 投稿 UI の特徴

- **「今日のメイク」投稿**: 複数商品をまとめてタグ付けし、一つのルック（メイクアップルック）として投稿可能
- **写真ファーストのUI**: 写真が大きく表示され、テキストは二次的
- **自動プロフィール表示**: 投稿者の肌質・年代・PC（パーソナルカラー）が自動でバッジ表示
- **リッチな写真編集**: 投稿時にフィルター、明るさ調整、トリミングが可能
- **「使い切りレポ」**: 商品を最後まで使い切った際の特別な投稿カテゴリ

#### コミュニティエンゲージメント

- **いいね**: ハートアイコン、ワンタップ
- **クリップ（保存）**: ブックマークアイコン、後で見返すため
- **コメント**: 投稿へのコメント + 返信のスレッド構造
- **フォロー**: ユーザー単位でフォロー可能
- **シェア**: LINE / Twitter / Instagram ストーリーへの共有

> **alche:me への示唆**: LIPSの「今日のメイク」投稿は、alche:me の日記（Beauty Log）機能と非常に近い。ただし LIPS は**他者に見せるための投稿**、alche:me は**自分のための記録**という違いがある。alche:me では「Quick Log」（写真+使用レシピの自動紐付け）の方向性が妥当。

### 2.6 商品詳細ページ設計

#### レイアウト構成（上から）

1. **商品画像**（公式画像 + ユーザー投稿のスウォッチ写真カルーセル）
2. **商品基本情報**（ブランド名 > 商品名 > 容量 > 価格）
3. **総合評価**（★平均 + クチコミ件数）
4. **カラーバリエーション選択**（カラーチップの横並び）
5. **クチコミ一覧**（フィルタ可能：年代・肌質・PC）
6. **関連商品**（「この商品を見た人はこれも見ています」）
7. **購入リンク**（EC サイトへの遷移ボタン）

#### 設計の優れた点

- **カラーバリエーション**: 色番ごとにクチコミが分かれており、自分が気になる色のレビューだけを見られる
- **写真の質重視**: ユーザー投稿のスウォッチ写真をフィルタリング可能（「実物写真のみ」等）
- **パーソナライズされたクチコミ**: 「あなたと同じ肌質の人のクチコミ」がハイライト表示
- **ブランドページへの遷移**: ブランド名タップでブランド内の全商品一覧へ

### 2.7 パーソナライゼーション

- **ユーザープロフィール**: 肌質、肌の色、パーソナルカラー（イエベ春/イエベ秋/ブルベ夏/ブルベ冬）、年代、肌悩み
- **おすすめフィード**: プロフィール情報 + 閲覧・いいね履歴に基づくアルゴリズム推薦
- **「あなたにおすすめ」セクション**: ホームフィード内に差し込まれる商品レコメンド
- **閲覧履歴**: 最近見た商品の一覧

### 2.8 お気に入り/クリップ管理

- **クリップ機能**: 投稿・商品をクリップ（ブックマーク）
- **クリップフォルダ**: カテゴリ別にクリップを整理可能（「リップ候補」「次買うもの」等）
- **マイページからアクセス**: クリップ一覧はマイページタブから

> **alche:me への示唆**: LIPS のクリップフォルダ機能は、alche:me の「Next Cosme（ウィッシュリスト）」と親和性が高い。フォルダ/タグによる分類（「リップ」「次のボーナスで」「夏用」等）を検討すべき。

### 2.9 モバイル特化デザインパターン

1. **スワイプ操作**: フィード内を左右スワイプでタブ切り替え
2. **プルダウンリフレッシュ**: フィード更新の標準パターン
3. **無限スクロール**: ページネーションなしの無限読み込み
4. **ボトムシート**: フィルター選択、商品詳細のクイックビュー
5. **大きなタップターゲット**: 最小 44x44pt のタッチエリア
6. **写真の全画面表示**: 写真タップで全画面プレビュー（ピンチズーム対応）
7. **スケルトンスクリーン**: ローディング中のプレースホルダー表示
8. **トーストメッセージ**: クリップ保存等のフィードバック

## 3. @cosme 詳細分析

### 3.1 アプリ概要

@cosme（アットコスメ）は、株式会社アイスタイルが運営する日本最大級のコスメ・美容の総合サイト/アプリ。1999年にPCサイトとして開始し、約42万点以上の化粧品データベースと1,800万件以上のクチコミを蓄積。@cosme STORE（実店舗+EC）も展開するオムニチャネルプラットフォーム。

### 3.2 タブ構成とナビゲーション

#### ボトムナビゲーション（5タブ構成）

| タブ | アイコン | 機能 |
|------|---------|------|
| **ホーム** | 🏠 | ニュース・特集・おすすめ |
| **ランキング** | 🏆 | ベストコスメ・カテゴリ別ランキング |
| **検索** | 🔍 | 商品・ブランド・クチコミ検索 |
| **マイページ** | 👤 | プロフィール・クリップ・購入履歴・ポイント |
| **ショッピング** | 🛒 | @cosme SHOPPING（EC） |

#### ナビゲーション設計の特徴

- **5タブ構成**だが、LIPS と比較して **ショッピング（EC）タブ**が独立しているのが特徴的
- 通知はタブではなくヘッダーのベルアイコン
- ホームタブは**マガジン型レイアウト**（特集記事・ランキングカルーセル・ニュースの混在フィード）
- **ハンバーガーメニュー**（ドロワー）はなし。全ての主要機能はタブから到達可能
- トップバーに**会員ランク**表示（@cosme メンバーシップ）

### 3.3 商品カタログ / ブラウジングパターン

#### カテゴリ階層（@cosme の最大の強み）

```
大カテゴリ
  ├─ ベースメイク
  │    ├─ ファンデーション
  │    │    ├─ リキッドファンデーション
  │    │    ├─ パウダーファンデーション
  │    │    ├─ クッションファンデーション
  │    │    └─ ...
  │    ├─ 化粧下地
  │    ├─ フェイスパウダー
  │    └─ コンシーラー
  ├─ アイメイク
  │    ├─ アイシャドウ
  │    │    ├─ パウダーアイシャドウ
  │    │    ├─ リキッドアイシャドウ
  │    │    └─ クリームアイシャドウ
  │    ├─ アイライナー
  │    ├─ マスカラ
  │    └─ アイブロウ
  ├─ リップ
  │    ├─ 口紅
  │    ├─ リップグロス
  │    ├─ リップティント
  │    └─ リップライナー
  ├─ チーク
  ├─ スキンケア
  │    ├─ クレンジング
  │    ├─ 洗顔料
  │    ├─ 化粧水
  │    ├─ 美容液
  │    ├─ 乳液
  │    ├─ クリーム
  │    └─ パック・マスク
  └─ ...（ヘアケア、ボディケア、ネイル、フレグランス、etc.）

ブランド階層:
  メーカー（親会社）
    └─ ブランド
        └─ 商品ライン（シリーズ）
            └─ 商品（SKU）
                └─ カラーバリエーション
```

#### ブラウジング UI パターン

1. **カテゴリグリッド**: ホームからアクセスできるアイコン付きカテゴリグリッド（3×4 程度）
2. **ブランドインデックス**: 50音順/アルファベット順のブランド一覧（サイドバーの文字インデックス付き）
3. **ランキングカルーセル**: 横スクロールのランキングカード（1位〜3位が大きく、4位以降はリスト）
4. **特集バナー**: 季節・イベント特集のバナーカルーセル（春コスメ特集、クリスマスコフレ等）
5. **商品比較機能**: 複数商品を並べて比較できるUI

> **alche:me への示唆**: @cosme のカテゴリ階層（大カテゴリ > 中カテゴリ > 小カテゴリ > ブランド > 商品 > 色番）は、alche:me の在庫管理における **商品階層ナビゲーション**（メーカー > ブランド > 商品ライン > カラー）の設計で参考にすべき最重要モデル。ただし、alche:me は「個人の在庫」が対象なので、@cosme のような網羅的カタログではなく、**ユーザーが所有するアイテムの中での階層ナビゲーション**として最適化する必要がある。

### 3.4 検索・フィルター UX

#### 検索機能

- **統合検索バー**: 商品名・ブランド名・成分名・キーワードをまとめて検索
- **バーコード検索**: 商品のバーコードをスキャンして商品ページへ直接遷移（JANコード対応）
- **サジェスト機能**: 検索キーワード入力中にリアルタイムでサジェスト表示
- **検索結果タブ**: 「商品」「クチコミ」「ブランド」「記事」で結果を切り替え
- **詳細検索**: 複数条件を組み合わせた絞り込み検索ページ

#### フィルター体系（@cosme の特筆すべき強み）

- **カテゴリ**: 大→中→小の段階的絞り込み
- **ブランド**: 検索 + 50音/ABC インデックス
- **価格帯**: スライダー or プリセット（〜1,000円 / 1,001〜3,000円 / 3,001〜5,000円 / 5,001円〜）
- **評価**: 星評価フィルター
- **クチコミ数**: 「クチコミ10件以上」等の閾値フィルタ
- **発売日**: 新着 / 過去半年 / 過去1年
- **投稿者属性**: 年代、肌質、肌の色
- **成分**: 特定成分の含有/非含有（「パラベンフリー」「ノンコメドジェニック」等）
- **受賞歴**: @cosme ベストコスメアワード受賞商品

> **alche:me への示唆**: @cosme のフィルター体系は網羅的だが、alche:me の文脈では「在庫フィルター」として応用する。例: カテゴリ別表示、開封日順、使用期限順、使用頻度順、ブランド別。AI レシピ検索では、テーマ（オフィス/デート/カジュアル）、所要時間、使用アイテム数でのフィルタが有効。

### 3.5 クチコミ・レビュー機能

#### クチコミの構造

```
クチコミ = {
  商品（1商品に対して1クチコミ）,
  総合評価（★0〜7、@cosme独自の7段階）,
  テキスト（詳細レビュー、自由記述）,
  写真（複数枚）,
  使用期間,
  購入場所,
  購入価格,
  リピート意向（「リピートしたい」「リピ買い中」等）,
  投稿者プロフィール（年代・肌質・肌色）,
  使用したカラーバリエーション
}
```

#### 特徴的な要素

- **7段階評価**: @cosme 独自。0〜7の星評価で、より細かい評価が可能
- **使用期間の記録**: 「1週間」「1ヶ月」「3ヶ月以上」等、使用期間を選択
- **購入場所の記録**: ドラッグストア、百貨店、EC、サンプル等
- **リピート情報**: 購買行動に直結する重要データ
- **クチコミの「参考になった」**: 有用なクチコミが上位に表示

> **alche:me への示唆**: @cosme の「使用期間」「リピート情報」は、alche:me の在庫管理で「開封日」「残量トラッキング」「PAO（開封後使用期限）」として応用済み。さらに「使用満足度」のトラッキングを日記機能に追加すれば、AIレシピの品質向上フィードバックループが作れる。

### 3.6 商品詳細ページ設計

#### レイアウト構成（上から）

1. **商品画像**（公式画像カルーセル）
2. **ブランド名 > 商品名**（ブランド名はタップでブランドページへ）
3. **価格・容量・発売日**
4. **総合評価**（★平均 + クチコミ件数 + ランキング情報）
5. **カラーバリエーション一覧**（カラーチップ + 色名）
6. **商品説明**（公式の商品説明テキスト）
7. **成分一覧**（全成分表示、折りたたみ展開）
8. **クチコミ一覧**（フィルタ・ソート可能）
9. **関連ランキング**（同カテゴリのランキング内での順位）
10. **購入リンク**（@cosme SHOPPING + 外部ECリンク）
11. **関連商品**（同ブランドの他商品、同カテゴリの人気商品）

#### 設計の優れた点

- **情報の網羅性**: 公式情報 + ユーザークチコミ + ランキング + 購入先を一画面に集約
- **成分表示**: 成分に敏感なユーザーへの対応（アレルギー成分のハイライト等）
- **クロスリンクの豊富さ**: ブランド→他商品、カテゴリ→ランキング、関連商品 → 回遊率の向上
- **メンバー限定情報**: ログインユーザーには「あなたと同じ肌質の評価」が表示

### 3.7 パーソナライゼーション

- **美容タイプ診断**: アプリ内の診断コンテンツ（肌質、パーソナルカラー診断）
- **マイベストコスメ**: ユーザーが自分のベストアイテムを選んで公開するリスト
- **おすすめ商品**: 閲覧・クチコミ・クリップ履歴に基づくレコメンデーション
- **会員ランク**: 投稿数・閲覧数等に応じたメンバーシップランク（プレミアム会員向けの先行レビュー等）
- **パーソナライズドランキング**: 「あなたの肌質で人気」のランキング表示

### 3.8 お気に入り/クリップ管理

- **クリップ機能**: 商品単位でクリップ（保存）
- **クリップ一覧**: マイページからアクセス、カテゴリ別にフィルタ可能
- **「欲しい」リスト**: 購入候補としてのウィッシュリスト機能
- **「持ってる」リスト**: 所有商品のマーキング（在庫管理の萌芽）

> **alche:me への示唆**: @cosme の「持ってる」リストは在庫管理の最も原始的な形。alche:me はここから大きく進化させ、AI による活用提案まで踏み込む点が差別化ポイント。

### 3.9 @cosme STORE 連携（EC統合）

- **アプリ内ショッピング**: @cosme アプリからシームレスに @cosme SHOPPING（EC）で購入可能
- **店舗連携**: @cosme STORE（実店舗）の在庫確認、商品取り置き
- **ポイント統合**: オンライン・オフラインのポイント統合
- **バーコードスキャン**: 実店舗で商品のバーコードをスキャンしてクチコミ確認
- **クーポン**: アプリ限定クーポンの配布

### 3.10 モバイル特化デザインパターン

1. **マガジンレイアウト**: ホームは雑誌のようなリッチなレイアウト
2. **スティッキーヘッダー**: 検索バーが常時表示（上スクロールで隠れ、下スクロールで再表示）
3. **タブ内セグメント**: 各タブ内でさらにセグメントコントロール（横スクロールタブ）
4. **カード型UI**: 商品・記事・クチコミすべてがカード形式
5. **ボトムシート**: フィルター選択、ソート順変更
6. **セクション区切り**: 明確なセクション見出し + 「もっと見る」リンク
7. **ローディング**: スケルトンスクリーン + プログレスインジケーター
8. **バッジ**: 通知バッジ、会員ランクバッジ、ベストコスメバッジ

## 5. alche:me への示唆と提言

### 5.1 ナビゲーション設計（FEAT-002 対応）

#### 提言 1: タブ構成の最適化

**現状の課題**: 5タブのうち「My Cosme」と「Next Cosme」が分離しており、コスメ管理が2タブに分散。一方で「設定/マイページ」がタブになく、LIPS/@cosme の標準パターンから外れている。

**提案A: コスメ統合型（推奨）**

| タブ | ラベル | 内容 |
|------|--------|------|
| 1 | ✨ AI美容部員 | チャット（コア体験） |
| 2 | 📦 マイコスメ | 在庫 + ウィッシュリスト（セグメントタブで切替） |
| 3 | 📖 レシピ | AI生成レシピ一覧 |
| 4 | 💕 日記 | Beauty Log |
| 5 | 👤 マイページ | プロフィール + 設定 + 統計 |

**理由:**
- LIPS/@cosme ともに「マイページ」タブが存在し、ユーザーの自然な期待に沿う
- 「My Cosme」と「Next Cosme」は「所有」と「欲しい」という同じ軸（コスメ管理）のセグメントであり、1タブ内のセグメントコントロールで切り替える方が情報アーキテクチャとして整合的
- 将来の SNS 機能追加時に、マイページタブにフォロー/フォロワー等を自然に追加できる
- スキャンは FAB（フローティングアクションボタン）として BottomNav の中央に配置するか、My Cosme 画面内のアクションとして配置

**提案B: スキャンFAB型**

| タブ | ラベル |
|------|--------|
| 1 | ✨ AI美容部員 |
| 2 | 📦 マイコスメ（在庫+ウィッシュ） |
| **中央FAB** | 📷 **スキャン** |
| 3 | 📖 レシピ |
| 4 | 💕 日記 |

- 設定/マイページは右上のアバターアイコンからアクセス（Instagram パターン）
- スキャンが最重要アクション（在庫登録のエントリーポイント）として中央 FAB に昇格

#### 提言 2: 商品階層ナビゲーション

### 5.2 日記機能の強化（FEAT-001 対応）

#### LIPS の「今日のメイク」投稿から学ぶべきこと

1. **写真ファースト**: テキストよりも写真が主役。alche:me の日記でも「まず写真を撮る」をエントリーポイントにすべき
2. **商品タグ付け**: 使用商品を写真にタグ付けする UI。alche:me では「使用レシピ」の自動紐付けとして実装
3. **Before/After**: ルック写真の前後比較。メイク前（素肌）→ メイク後の変化が可視化されると日記としての価値が上がる
4. **テンプレート**: 「通勤メイク」「デートメイク」等のプリセットで入力負荷を下げる

#### @cosme の「使用期間」トラッキングから学ぶべきこと

1. **使用期間の可視化**: アイテムごとに「いつから使っているか」「どのくらい使ったか」が分かる
2. **リピート情報**: 「リピ買い」の概念。alche:me では「使い切った → 同じ商品を再購入」のフローとして実装可能

### 5.3 商品 × ルック（レシピ）の関係性表示

#### LIPS のパターン: 商品 → ルック

LIPS では、商品詳細ページからその商品を使ったユーザーの投稿（＝ルック）が見られる。

#### @cosme のパターン: 商品 → クチコミ → 他商品

@cosme では、クチコミ内で「一緒に使った商品」への言及があり、関連商品への回遊が生まれる。

#### alche:me での応用

```
商品詳細ページ:
  ├─ 基本情報（ブランド、カテゴリ、色、残量、PAO）
  ├─ 使用履歴（日記との紐付け → カレンダーヒートマップ）
  ├─ 登場レシピ一覧（この商品を使ったAIレシピ）
  ├─ 相性の良い組み合わせ（AIが分析した、よく一緒に使われる商品）
  └─ 代替候補（同カテゴリ・近い色味の手持ちアイテム）

レシピ詳細ページ:
  ├─ レシピ情報（テーマ、ステップ、所要時間）
  ├─ 使用アイテム一覧（タップで商品詳細へ）
  ├─ 再現した日の日記リンク
  ├─ 類似レシピ（同テーマ/同アイテムの別レシピ）
  └─ アレンジ提案（「このレシピ + 1アイテムで別の雰囲気に」）
```

### 5.4 パーソナライゼーションの強化

#### LIPS/@cosme が実現していないこと（alche:me の機会領域）

1. **在庫ベースのパーソナライゼーション**: LIPS/@cosme は「世の中にある全商品」からのレコメンド。alche:me は「手持ちの商品」からの提案であり、100% 実現可能なレコメンドを提供できる
2. **使用パターン分析**: 過去の日記データから「この曜日はこのメイクが多い」「気分が良い日はこの口紅を使う傾向」等のインサイトを提供
3. **消費予測**: 使用頻度と残量から「来月なくなりそうなアイテム」を予測し、Next Cosme（ウィッシュリスト）に事前提案
4. **マンネリ検知**: 「最近2週間同じリップしか使っていません。こんなレシピはどうですか？」
5. **季節対応**: 季節 × 肌状態 × 手持ち在庫で最適なレシピを自動提案

### 5.5 写真/画像ハンドリングの改善

#### LIPS から学ぶべき写真 UX

1. **スウォッチ表示**: 商品の実際の発色を腕や唇で確認できる写真。alche:me ではスキャン時に自動で撮影した写真を活用
2. **ライティングガイド**: 写真撮影時に「自然光で撮ると色が正確に写ります」等のガイド表示
3. **写真グリッド**: 日記のタイムライン表示で、ルック写真のグリッド表示（Instagram のプロフィールページのようなパターン）

### 5.6 日本の美容市場で成功している UX の共通要素

#### 両アプリに共通する成功要因

1. **「クチコミ」文化への最適化**: 日本のユーザーは購入前にクチコミを徹底的に調べる。alche:me では、AI が「クチコミ」的な役割を果たし、「この商品の使い方で最高の結果を出す方法」を提案する
2. **パーソナルカラー/肌質の重視**: 日本の美容市場ではPC（イエベ/ブルベ）と肌質が非常に重要。両アプリともプロフィールの中核要素としている
3. **プチプラ vs デパコスの価格帯意識**: 日本のユーザーは価格帯に敏感。alche:me でも「手持ちのプチプラでデパコス風に」等の訴求が効果的
4. **季節感**: 桜コスメ、サマーコレクション、クリスマスコフレ等、季節に紐づいたコスメ文化。AI レシピも季節感を取り入れるべき
5. **「使い切り」美学**: 物を大切にする日本文化。LIPS の「使い切りレポ」は人気コンテンツ。alche:me の在庫管理で「使い切りおめでとう」演出は非常に効果的
6. **カワイイ UI**: 角丸、パステルカラー、アイコンの使用。alche:me の現在のデザインシステム（ローズ、ゴールド、クリーム）は日本の美容アプリの文脈で適切
7. **スクロールの快適さ**: 日本のユーザーは長いページのスクロールに慣れている（@cosme の商品ページは非常に長い）。ただし重要なアクション（CTA）はファーストビューに配置
