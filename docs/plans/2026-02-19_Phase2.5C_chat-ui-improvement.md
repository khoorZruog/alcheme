### 9B. `QuickActionChips` データ駆動化

**`components/quick-action-chips.tsx`** — 内部ロジック書き換え（外部 API は同じ）

#### 設計方針（ユーザーフィードバック反映）

**問題**: ハードコードされた季節/イベントチップ（"バレンタインメイク"、"冬のツヤ肌メイク"）は:
- 時期外れのリスク（バレンタイン後にまだ表示される等）
- マンネリ感（"またツヤ肌か"）

**解決**: 全チップを**データ駆動の固定ラベル**にする。ラベルは変わらないが、裏側で実データが動くため結果が毎回異なる。季節名やイベント名をハードコードしない。

#### チップ構成（4つ固定）

| # | ラベル | 根拠 |
|---|--------|------|
| 1 | "おまかせ提案" | concierge が `get_today_context` で季節・在庫・トレンドを考慮して3テーマ提案。毎回異なる結果 |
| 2 | "手持ちコスメで新しい組み合わせ" | 在庫データが毎回変わるためマンネリにならない |
| 3 | "眠ってるコスメ活用" | `get_today_context` の underused_items に基づき毎回異なる |
| 4 | "今日の天気に合うメイク" | agent が `get_weather()` で現在の天気を取得して提案。ラベルは固定だが天気が変われば結果は変わる |

全チップとも `useWeather` 不要。天気チップも固定ラベルで、天気取得は agent 側で行う。

## Batch 10: Stop Button + Retry/Regenerate

### 10A. `useChat` に `stopGeneration` / `retryLastMessage` 追加

**`hooks/use-chat.ts`**

```typescript
// 生成停止
const stopGeneration = useCallback(() => {
  abortRef.current?.abort();
  abortRef.current = null;
  setIsLoading(false);
  setThinkingStatus(null);
  // ストリーミング中メッセージを確定
  setMessages(prev => prev.map(m =>
    m.is_streaming ? { ...m, is_streaming: false } : m
  ));
}, []);

// リトライ（最後のアシスタントメッセージを削除して再送）
const retryLastMessage = useCallback(() => {
  const lastUser = [...messages].reverse().find(m => m.role === "user");
  if (!lastUser) return;
  setMessages(prev => {
    const idx = prev.findLastIndex(m => m.role === "assistant");
    return idx > 0 ? prev.slice(0, idx) : prev;
  });
  sendMessage(lastUser.content);
}, [messages, sendMessage]);
```

AbortError catch を改善: ストリーミング中メッセージの `is_streaming` を false に、受信済みコンテンツは保持。

## 設計判断

1. **チップ = データ駆動の固定ラベル4つ** — 季節名やイベント名をハードコードしない。全チップのラベルは固定だが裏側で実データ（在庫・天気・トレンド）が動くため毎回異なる結果が返る。天気チップのラベルも "今日の天気に合うメイク" で固定（agent 側で天気取得）。`useWeather` フック不要
2. **Stop ボタン = 送信ボタン変形** — 新しいフローティング要素を追加せず、既存の送信ボタンをインプレース変更（ChatGPT/Claude と同じパターン）
3. **リトライ = 最後のメッセージのみ** — グローバル UI ではなく、最後のアシスタントメッセージ直下にインライン表示
4. **リッチカード = 新 SSE イベント型** — テキストからのパース（fragile）ではなく、ツール関数レスポンスを傍受して構造化データとして送信。既存の recipe_card パターンに準拠
5. **Batch 9/10 は独立** — 並行実装可能。Batch 11 → 12 は依存関係あり
