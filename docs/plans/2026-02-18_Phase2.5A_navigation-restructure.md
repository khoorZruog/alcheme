## Context

alche:me の現在の5タブ構成（AI美容部員 / My Cosme / Recipe / Next Cosme / 日記）は Phase 1 で構築されたが、11本のDeep Research（LIPS, @cosme, TikTok, Instagram, WEAR, Duolingo, Apple Journal, Sephora, AI Chat UI, Generative UI, 新興美容アプリ）の知見に基づき、以下を実施する:

1. **タブ構成の再設計** — My Cosme + Next Cosme 統合、中央「+」ボタン、マイページ新設
2. **中央「+」ボタン** — 在庫登録ではなく、日々のエンゲージメント（Beauty Log）を中心としたクイックアクション
3. **マイページ** — Profile + Beauty Log 履歴 + Settings 導線の統合

### ユーザーフィードバック反映
- 在庫登録は利用継続につれて頻度が下がる → 中央「+」は別のエントリポイントに
- Beauty Log はコア機能 → マイページだけでなく中央「+」からもアクセス可能に
- マイページ構成は Deep Research 知見に基づいて設計

### Gemini (PO/CTO) レビュー追加アドバイス（承認済み）
- **Quick Action Sheet の起動速度**: 最も押されるボタン。遅延読み込みしすぎない
- **触覚フィードバック**: ボタンタップ時に HapticFeedback（PWA の `navigator.vibrate`）
- **"Move" アクション**: 「欲しい」→「持ってる」へのワンタップ移動（購入完了トリガー）
- **ストリーク可視化**: マイページ Data Stats に連続記録日数を目立つ位置に
- **Recipe タブ**: 「お気に入り」クイックフィルタで過去の成功レシピを即呼び出し
- **SWR キャッシュ**: セグメント切替時に `keepPreviousData: true` で瞬時切替
- **Liquid Glass**: 中央「+」ボタンに透明感あるグラデーション + 内部光沢

## リスク対策

| リスク | 対策 |
|--------|------|
| `/suggestions` ブックマーク破損 | リダイレクトページとして残す |
| inventory/page.tsx の肥大化 | ビューコンポーネントを分離（have/want） |
| Beauty Log の発見性低下 | 中央「+」+ マイページプレビューで2箇所の導線確保 |

| マイページの初回ロード性能 | SWR + Suspense boundaries + スケルトン表示 |
